# AI Memory

这个目录用于沉淀会在后续 AI 任务中反复用到、但容易在上下文压缩后丢失的项目信息。

## 开工前必读

1. 先读本文件，再按任务需要读取其他文件。
2. 完成任何非微小代码改动后，同步更新至少一个相关 memory 文件。
3. 记录重点放在长期有效的信息、关键决策和后续风险，不写流水账。

## 文件导航

- `product-context.md`：产品规则、用户长期偏好、容易遗漏的细节需求。
- `engineering-context.md`：工程结构、实现边界、推荐模式、已知坑点。
- `change-log.md`：按时间倒序记录重要 AI 改动与原因。
- `open-questions.md`：尚未闭环的事项、风险、待确认点。

## 当前项目速记

1. AIClientCore 是一个基于 Tauri v2 + Vue3 + TypeScript 的桌面 AI 聚合客户端。
2. 前端使用 UnoCSS 做样式方案，Pinia 做状态管理，Vue Router 做路由。
3. 后端通过 Rust 调用 Tauri API，负责全局快捷键、窗口管理、系统主题监听等。
4. 项目已经在 `.agents/skills/` 下维护多类工程 skill，本目录用于补足"长期项目记忆"，避免只靠对话上下文。
5. 后续 AI 在实现、重构、修复、评审后，应同时判断是否需要更新这里的记忆文件。
